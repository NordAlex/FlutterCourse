Everything you can place in a variable is an object, and every object is an instance of a class. Even numbers, functions, and null are objects. All objects inherit from the Object class.

Although Dart is strongly typed, type annotations are optional because Dart can infer types. When you want to explicitly say that no type is expected, use the special type dynamic.

Dart supports generic types, like List<int> (a list of integers) or List<dynamic> (a list of objects of any type).

The entry point of a Dart class is the main() function. This method acts as a starting point for Flutter apps as well.

Dart supports top-level functions (such as main()), as well as functions tied to a class or object (static and instance methods, respectively). You can also create functions within functions (nested or local functions).

Similarly, Dart supports top-level variables, as well as variables tied to a class or object (static and instance variables). Instance variables are sometimes known as fields or properties.

Dart doesn't have the keywords public, protected, and private. If an identifier starts with an underscore (like _firstname), it's private to its library.

Identifiers can start with a letter or underscore (_), followed by any combination of those characters plus digits.

Dart has both expressions (which have runtime values) and statements (which don't). For example, the conditional expression condition ? expr1 : expr2 has a value of expr1 or expr2. Compare that to an if-else statement, which has no value. A statement often contains one or more expressions, but an expression can't directly contain a statement.

print('Welcome to Dart');

Dart supports 2 types of numbers: integers and doubles. There is a special type num which might contain integer or double.

int a = 8;
double b = 3.5;
num c = 34; // integer or double

Dart support strings:

String str = 'Dart';

And booleans:

bool flag = false;

You may skip the type definition as Dart is smart enough to get it automatically. There are 3 keywords to describe the mutability of the variable.
Use final when you don't plan to change the value later on. Prefer const when the value is known during complilation. Otherwise use var.

You may check the type by using runtimeType property.

final x = 2;
const y = 2.3;
var z = 'Flutter';

print(x.runtimeType); // int
print(y.runtimeType); // double
print(z.runtimeType); // String

List is the one of the most used primities in Dart. The compiler is smart enough to get the type of the list elements automatically.

final a = [1,2,3];
final b = ['hello', 'world'];
final c = [1, 'abc', 2.1];

Though it is a best practice to specify the type by yourself, like this:

final generic = <int>[1,2,3];

Map is commonly used for small data structures, or when the data model is unkown.

final m = {'firstname': 'John', 'lastname': 'Smith', 'age': 20.3};
print(m);
print(m.keys);
print(m.values);
print(m['firstname']);

When the model is known, prefer classes.

class Person {
    String firstname;
    String lastname;
    double age;
}

Always prefer const constructors and final properties in classes:

class Person {
    const Person(this.firstname, this.lastname, this.age);

    final String firstname;
    final String lastname;
    final double age;
}

Functions definition looks like this:

int multiply(int a, int b) {
    return a * b;
}
